# PRD: Formcast

## 목표
개발자가 JSON Schema를 붙여넣으면, 그 스키마가 만들어내는 폼과 검증 동작을 즉석에서 확인·테스트할 수 있게 한다.

## 사용자
JSON Schema로 API 요청/설정을 정의하는 개발자. 스키마가 실제로 어떤 폼·검증을 만들어내는지 빠르게 확인하고 싶은 사람. (폼을 채우는 최종 사용자가 아니라, 스키마를 작성하는 개발자가 대상이다.)

## 핵심 기능
1. 좌우 2분할 화면 — 왼쪽 스키마 편집기(텍스트), 오른쪽 실시간 폼.
2. 스키마를 수정하면 오른쪽 폼이 즉시 재렌더된다.
3. 폼 입력값을 실시간 검증하고, 각 필드 밑에 에러 메시지를 표시한다.
4. 검증된 결과를 폼 하단에 JSON으로 보여주고, 복사 버튼을 제공한다.
5. 진입 시 예제 스키마 프리셋(2~3개)을 제공해 빈 화면을 없앤다.

## 지원하는 JSON Schema 서브셋 (MVP 범위)
- 타입: `string`, `number`, `boolean`, `enum`(select로 렌더)
- 제약: `required`, `minLength`, `maxLength`, `minimum`, `maximum`, `pattern`
- 메타: `title`(라벨), `description`(힌트 텍스트)
- 평면(1뎁스) 객체 스키마만 지원한다.

## MVP 제외 사항
- 중첩 객체(`object` 안의 `object`), 배열(`array`)
- `$ref`, `allOf` / `oneOf` / `anyOf`, 조건부 스키마
- 스키마를 GUI로 그리는 빌더 (스키마는 텍스트로 입력)
- 데이터 영속화, 로그인, 공유 링크

## 에러 처리 (2층)
- 스키마 파싱 에러 (잘못된 JSON, 미지원 타입) → 왼쪽 패널에 표시
- 값 검증 에러 (제약 위반) → 오른쪽 각 필드 밑에 표시

## 디자인
- 도구처럼 보여야 한다. 마케팅 페이지가 아니라 매일 쓰는 개발자 대시보드.
- 무채색 기반 + 시맨틱 색(성공/에러)만 포인트로. 좌측 정렬, 고밀도 레이아웃.
